# 网页导出与本地游玩

本项目使用 Godot 4 的 `Web` 导出预设。网页成品放在 `build/web/`，该目录是生成物，不提交到 Git。

## 第一次准备

1. 使用 Godot 打开项目根目录下的 `project.godot`，等待资源导入完成。
2. 确认已经安装与当前 Godot 版本一致的 Web Export Templates。
3. 项目必须保留 `assets/fonts/MindscapeWebFont.tres` 引用的 CJK 与 Symbols 2 字体，并在 `project.godot` 中通过 `gui/theme/custom_font` 使用这个组合字体。网页不能可靠使用电脑系统字体；缺少内置字形时，中文、盲文或 UI 符号会显示成方框。

## 导出网页版本

在项目根目录执行：

```bash
mkdir -p build/web
godot --headless --path . --export-release Web build/web/index.html
cp assets/fonts/OFL.txt build/web/NotoSansCJK-OFL.txt
```

如果终端找不到 `godot`，可以在 Godot 编辑器中选择 `Project > Export > Web > Export Project`，输出文件选择 `build/web/index.html`。

每次修改游戏代码、场景、图片、声音或字体后，都要重新导出。部署或分享时必须上传 `build/web/` 中的全部文件，不能只上传 `index.html`。

## 在本地电脑游玩

不能直接双击 `index.html`。在项目根目录启动本地 HTTP 服务器：

```bash
python3 -m http.server 8765 --bind 127.0.0.1 --directory build/web
```

然后用浏览器打开：

```text
http://127.0.0.1:8765
```

保持运行命令的终端窗口开启。结束游戏服务器时，在该终端按 `Control + C`。

如果重新导出后浏览器仍显示旧版本，按 `Command + Shift + R` 强制刷新，或关闭旧标签页后重新打开。

## 中文字体回归检查

字体配置或 UI 改动后，可以执行：

```bash
godot --headless --rendering-method gl_compatibility --audio-driver Dummy --path . tests/WebFontTest.tscn
```

看到下面的输出代表项目内置字体存在，并包含测试所需的中文、盲文和 UI 符号：

```text
PASS: bundled web fonts cover required Chinese, Braille, and UI symbols
```

## 部署到服务器

1. 重新导出，并确认本地版本能够启动。
2. 将 `build/web/` 内的所有文件上传到同一个网站目录。
3. 正式网站使用 HTTPS。
4. 服务器需要以 `application/wasm` 类型提供 `.wasm` 文件。
5. 打开公网网址，检查登录、存档、移动、交互、视角切换、声音和中文显示。

当前导出预设关闭了线程支持，因此不要求服务器配置跨源隔离响应头。以后如果开启 Godot Web 的线程支持，需要同时配置 `Cross-Origin-Opener-Policy` 和 `Cross-Origin-Embedder-Policy`。

浏览器存档保存在该网站域名对应的浏览器数据中。更换域名、清理浏览器数据或使用无痕模式，都可能看不到原有存档。

## 中国内地部署（推荐方案）

如果需要中国内地用户稳定访问，推荐使用“腾讯云中国内地 COS + 自定义域名 + ICP 备案 + 中国境内 CDN”。阿里云 OSS + CDN 也可以，原理相同。不要把 itch.io、GitHub Pages、Vercel 或香港服务器当作中国内地稳定访问的正式方案。

### 推荐架构

```text
玩家浏览器 -> 已备案的 HTTPS 域名 -> 中国境内 CDN -> 广州或上海 COS 存储桶
```

### 准备条件

1. 腾讯云实名认证账号。
2. 一个自有域名。
3. 在腾讯云中国内地地域创建 COS 存储桶，例如广州或上海。
4. 为域名完成 ICP 备案。中国境内 CDN 不能加速未备案域名。
5. 如果游戏要面向公众正式运营或收费，ICP 备案不等于游戏出版运营许可，还需要另行确认网络游戏审批、运营资质、实名和未成年人保护等合规要求。

### COS 与 CDN 配置步骤

1. 在 COS 控制台创建中国内地存储桶。
2. 开启“静态网站”，索引文档填写 `index.html`，并开启强制 HTTPS。
3. 将 `build/web/` 里的所有文件上传到存储桶根目录。
4. 检查文件响应类型：
   - `.html`：`text/html`
   - `.js`：`application/javascript`
   - `.wasm`：`application/wasm`
   - `.pck`：`application/octet-stream`
5. 绑定已经备案的自定义域名和 HTTPS 证书。
6. 在腾讯云 CDN 中添加该域名，加速区域选择“中国境内”，源站选择 COS 静态网站源站。
7. 按控制台给出的地址配置 DNS CNAME。
8. 测试阶段把 CDN 缓存时间设短一些；每次发布新版本后执行 CDN URL/目录刷新，避免玩家加载旧的 `index.pck`。

当前网页包约为 111MB。首次打开时每名玩家可能需要下载接近这个量级的数据；100 次完整首次加载约产生 11GB 流量，1000 次约产生 111GB 流量，实际数值会受浏览器和 CDN 缓存影响。正式推广前应购买合适的 COS/CDN 流量包，并继续压缩图片、音频和字体资源。

### 暂时没有备案时

可以先用腾讯云香港 COS、香港轻量服务器或海外托管做小范围测试，这些方案通常不要求中国内地 ICP 备案。但是数据仍需要经过跨境链路，速度和可用性会随地区、运营商和时间变化，不能承诺中国内地稳定访问。正式上线仍应迁移到中国内地地域并完成备案。

## 常见问题

- 页面打不开：确认本地服务器命令仍在运行，并访问 `http://127.0.0.1:8765`。
- 文字或符号显示方框：确认组合字体引用的字体文件存在，重新等待 Godot 导入资源，再重新导出全部网页文件。
- 页面一直加载：打开浏览器开发者工具查看 Console 和 Network；重点检查 `.pck`、`.wasm` 是否返回 200。
- 服务器上能打开但资源缺失：重新上传整个 `build/web/`，不要改动导出文件名。
- 修改后没有变化：重新导出并强制刷新浏览器缓存。
