本文整理当前项目里已经落地的关卡逻辑、对应视角需求、现成美术/截图线索，以及每一关想表达的设计方向。

## 1. 游戏总结构

核心是一个连续大地图的横版探索解谜游戏。玩家从中央广场出发，逐步解锁四种特殊视角，再回到中心完成结局。

### 已实现视角

- 普通视角：基础探索、对话、机关互动
- 盲人视角：全黑视野 + 可视圈 + 听觉与触觉感知
- ADHD 视角：高速移动、冲刺、强调节奏和操作
- 自闭症视角：识别细节、模式、隐藏提示
- 抑郁视角：灰暗感知、潜台词、隐藏答案、情绪痕迹

### 全局设计理念

- 不是“某种视角更正确”，而是每种视角都能帮助玩家看到不同层面的世界
- 谜题不靠纯文字密码，而靠观察、尝试、空间记忆和视角切换
- 关卡之间互相返回，形成银河恶魔城式的回访和补完
- 叙事尽量通过环境、NPC 闲聊和记忆片段传递，不用长篇说教

## 2. 关卡总览

### 2.1 纹理墙 / 石门

- 位置：森林侧主线前段
- 需要视角：盲人视角
- 当前逻辑：玩家靠近石墙后，切到盲人视角，用按键触摸不同“凸起石块”，根据方向/距离反馈找到正确顺序
- 玩法重点：
  - 必须先进入盲人视角
  - 通过“触摸感”而不是视觉阅读来解谜
  - 正确按键会让对应石块沉下去，四步完成后石门开启
- 设计理念：
  - 把“看不见”变成“用触觉和方向感理解结构”
  - 让玩家把盲人视角理解为一种有效感知，而不是惩罚

### 2.2 找不同密室

- 位置：森林主线中段
- 需要视角：普通 + ADHD + 抑郁 + 自闭，玩家在不同视角间对比
- 当前逻辑：
  - 房间内 5 个物体在不同视角下呈现不同状态
  - 玩家需要在多个视角中比对，推理出“两个视角一致”的正确状态
  - 不是靠单一答案，而是靠观察差异
- 玩法重点：
  - 视角切换本身就是解谜动作
  - 玩家要记住不同视角看到的状态，再回到普通视角作答
- 设计理念：
  - 强调“同一世界，多个解释”
  - 让视角差异成为推理工具

### 2.3 宴会厅油画

- 位置：森林后段
- 需要视角：自闭症视角、抑郁视角
- 当前逻辑：
  - 油画里会看到跳舞的小人
  - 玩家需要看懂舞步顺序，再在外部机关里复现
  - 这关偏向“模式识别 + 顺序记忆”
- 玩法重点：
  - 不是阅读说明，而是看懂图像中的规律
  - 自闭视角更容易看见结构、节奏和重复
- 设计理念：
  - 把“细节识别”和“模式感知”做成优势，而不是附加负担

### 2.4 石台拼图 / 九宫格记忆谜题

- 位置：水坝区域
- 需要视角：抑郁视角，且在通关前就会周期性闪现正确答案
- 当前逻辑：
  - 玩家推动或摆放九宫格碎片

- 玩法重点：
  - 拼图本身是空间整理，不是读密码
  - 抑郁视角负责“把隐藏的图样显露出来”
- 设计理念：
  - 让“被忽略的痕迹”变成可见信息
  - 让抑郁视角成为看见完整图像的方式，而不是负面滤镜

### 2.6 NPC 密码台

- 位置：许愿堂 / 天文台前段
- 需要视角：抑郁视角 + 自闭症视角
- 当前逻辑：
  - 5 个 NPC 各说一句话
  - 普通模式下是闲聊
  - 抑郁视角会显示潜台词
  - 自闭症视角更容易读出密码本/标记信息
  - 玩家按顺序提取词语形成最终答案
- 玩法重点：
  - 不是“找字谜”，而是读懂人没说出口的那部分
  - 通过人物语气、潜台词和语义串联来推理
- 设计理念：
  - 将沟通差异转化为可观察的文本层次
  - 让“理解他人”成为主线的一部分

## 3. 合作型关卡

这些关卡的重点不是某一个视角单独通关，而是视角间的合作。

### 3.1 水坝

- 需要视角：盲人 + 聋人
- 玩法：
  - 盲人视角判断异常水流、管道方向、共振位置
  - 聋人视角观察机械振动、齿轮同步和平台节奏
  - 两者合起来恢复水轮机
- 核心体验：
  - 一边“听结构”，一边“看震动”
  - 两种感觉拼成完整解决方案
- 相关素材：
  - [`assets/houses/dam_workshop_front.png`](/Users/xiaoxian/Documents/New Game/assets/houses/dam_workshop_front.png)
  - [`assets/houses/dam_workshop_back.png`](/Users/xiaoxian/Documents/New Game/assets/houses/dam_workshop_back.png)

### 3.2 天文台

- 需要视角：盲人 + 聋人 + ADHD
- 玩法：
  - 盲人负责共振频率与路线
  - 聋人负责齿轮和机械规律
  - ADHD 负责高速平台穿越和镜片调整
  - 三种视角一起启动望远镜
- 核心体验：
  - “看见星星”前先理解系统如何协同
  - 让不同视角各自承担一部分信息
- 相关素材：
  - [`assets/houses/old_dance_hall_front.png`](/Users/xiaoxian/Documents/New Game/assets/houses/old_dance_hall_front.png)
  - [`assets/houses/old_dance_hall_back.png`](/Users/xiaoxian/Documents/New Game/assets/houses/old_dance_hall_back.png)

### 3.3 地下管网 / 地下迷宫

- 需要视角：全视角轮流合作
- 玩法：
  - 盲人：靠风声、路线音效和触觉确认方向
  - 聋人：看振动和轨道节奏
  - ADHD：冲刺穿越限时区域
  - 抑郁：看见遗漏的痕迹与被忽略的路径信息
- 核心体验：
  - 地下部分更像“综合判断”
  - 不是单一视角的专属关卡，而是组合使用前面学到的能力

## 5. 角色与视角对应关系

项目里并不是把角色硬写成“某种障碍”，而是让不同视角成为不同的感知方式。

### 四种核心体验

- 盲人：用听觉、触摸和方向感理解空间
- 抑郁：用痕迹、潜台词和被忽略的信息重建完整感受
- 自闭：更容易发现模式、结构、对齐与细节
- ADHD：更擅长快速行动、冲刺与节奏性切换

### 相关 NPC 和提示

- 盲文学者：教盲文、教触感阅读
- 手语少女：只在对应视角下更清晰地交流
- 长椅老人：贯穿全局的记忆锚点

## 7. 设计原则总结

1. 先让玩家“看见并愿意走下去”
2. 再让他们“感受到不同视角不是惩罚，而是工具”
3. 谜题尽量建立在空间、声音、纹理、节奏和观察上
4. 让 NPC 和环境讲故事，少用生硬说明
5. 所有视角都应让玩家更接近“理解世界”，而不是单纯制造阻碍
